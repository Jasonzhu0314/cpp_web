## 多客户端处理
为了处理多个客户端的连接服务端，使用以下方式进行处理
### 多个子线程
每当有一个新的客户端连接到服务端是，创建一个新的子线程处理客户端的连接，服务端则可以继续等待其他新的客户端连接
```c++
(1) 服务端等待accept，阻塞线程
(2) 客户端连接，服务端创建子线程处理
(3) 服务端回到步骤(1)，继续执行
```
优点：可以同时连接多个客户端

缺点：每当一个客户端请求连接时，就创建一个线程，关闭时，销毁线程，当处于高并发时，频繁的创建和销毁线程，影响效率
